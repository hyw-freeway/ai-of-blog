## 一、项目架构概览

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 19 + Vite | 单页应用，构建后生成静态文件 |

## 二、部署前准备

### 2.1 购买服务器

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/lighthouse)
2. 购买轻量应用服务器（推荐配置：2核2G 或更高）
3. 选择系统镜像：**OpenCloudOS** 或 **CentOS 8**
4. 设置 root 密码

### 2.2 获取服务器信息

- **公网 IP**：如 `106.55.56.92`
- **root 密码**：购买时设置或后续重置

### 2.3 本地准备

- 确保项目代码已推送到 GitHub
- 安装 SSH 客户端（Windows 自带 PowerShell 或使用 PuTTY）
- 安装 SFTP 工具（可选，如 WinSCP、FileZilla）

## 三、服务器初始化

### 3.1 SSH 连接服务器

```bash
ssh root@你的服务器IP
```

首次连接会提示确认服务器指纹，输入 `yes` 继续，然后输入密码。

> **注意**：输入密码时屏幕不会显示任何字符，这是 Linux 安全机制。

### 3.2 更新系统

```bash
# OpenCloudOS / CentOS 使用 dnf
dnf update -y
dnf install -y curl wget git vim unzip
```

## 四、安装 Node.js

### 4.1 安装方式选择

由于 OpenCloudOS 的特殊性，标准的 NodeSource 脚本可能无法使用，推荐使用**二进制包直接安装**。

### 4.2 下载并安装 Node.js 18

```bash
# 进入临时目录
cd /tmp

# 从国内镜像下载（速度快）
wget https://npmmirror.com/mirrors/node/v18.20.2/node-v18.20.2-linux-x64.tar.xz

# 解压
tar -xf node-v18.20.2-linux-x64.tar.xz

# 移动到系统目录
mv node-v18.20.2-linux-x64 /usr/local/node

# 配置环境变量
echo 'export PATH=/usr/local/node/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 验证安装
node -v  # 应显示 v18.20.2
npm -v
```

### 4.3 配置 npm 国内镜像

```bash
npm config set registry https://registry.npmmirror.com
```

### 4.4 安装 PM2

```bash
npm install -g pm2
```

**PM2 的作用**：
- 保持 Node.js 应用持续运行
- 崩溃后自动重启
- 支持开机自启
- 提供日志管理

## 五、安装并配置 MySQL

### 5.1 安装 MySQL

```bash
dnf install -y mysql-server

# 启动并设置开机自启
systemctl start mysqld
systemctl enable mysqld

# 查看状态
systemctl status mysqld
```

### 5.2 创建数据库和用户

```bash
# 登录 MySQL（OpenCloudOS 默认 root 无密码）
mysql -u root
```

执行以下 SQL：

```sql
-- 创建数据库
CREATE DATABASE blog_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建专用用户（替换密码）
CREATE USER 'blog_user'@'localhost' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON blog_db.* TO 'blog_user'@'localhost';
FLUSH PRIVILEGES;

-- 切换到 blog_db
USE blog_db;

-- 创建 admin 表（管理员）
CREATE TABLE admin (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    createdAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建 articles 表（文章）
CREATE TABLE articles (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255) NOT NULL,
    content LONGTEXT NOT NULL,
    tags VARCHAR(255) DEFAULT '',
    author VARCHAR(50) NOT NULL,
    ai_summary TEXT,
    embedding JSON,
    createTime TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updateTime TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 插入默认管理员
INSERT INTO admin (username, password) VALUES ('admin', '123456');

-- 退出
exit;
```

> **重要**：表字段名必须与后端代码期望的一致，否则会导致 500 错误。

## 六、部署项目代码

### 6.1 克隆代码

```bash
mkdir -p /var/www
cd /var/www

# 从 GitHub 克隆
git clone https://github.com/你的用户名/仓库名.git
cd 仓库名
```

> 如果 GitHub 访问慢，可使用 SFTP 工具直接上传代码。

### 6.2 配置后端

```bash
cd /var/www/仓库名/back_end_of_blog

# 安装依赖
npm install

# 创建环境变量文件
cat > .env << 'EOF'
# 服务器配置
PORT=3001

# MySQL 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=blog_user
DB_PASSWORD=你的数据库密码
DB_NAME=blog_db

# JWT 密钥
JWT_SECRET=your_jwt_secret_key_change_this
JWT_EXPIRES_IN=7d

# AI 配置（如有）
AI_PROVIDER=deepseek
AI_API_KEY=你的AI_API_KEY
AI_BASE_URL=https://api.deepseek.com/v1
AI_MODEL=deepseek-chat
EOF
```

### 6.3 启动后端

```bash
# 使用 PM2 启动
pm2 start app.js --name blog-api

# 保存进程列表
pm2 save

# 设置开机自启
pm2 startup
# 执行它输出的命令
```

### 6.4 构建前端

```bash
cd /var/www/仓库名/front_end_of_blog

# 修改 API 默认地址（重要！）
sed -i "s|'http://localhost:3001'|''|g" src/services/api.js

# 或者创建环境变量文件
echo "VITE_API_BASE_URL=http://你的服务器IP" > .env.production

# 安装依赖
npm install

# 构建生产版本
npm run build
```

构建完成后，`dist` 目录包含所有静态文件。

## 七、配置 Nginx 反向代理

### 7.1 安装 Nginx

```bash
dnf install -y nginx
systemctl start nginx
systemctl enable nginx
```

### 7.2 创建站点配置

```bash
cat > /etc/nginx/conf.d/blog.conf << 'EOF'
server {
    listen 80;
    server_name 你的服务器IP;

    # 前端静态文件
    location / {
        root /var/www/仓库名/front_end_of_blog/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # SSE 流式响应支持
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 86400s;
    }

    # 上传文件访问
    location /uploads {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
    }

    # 文件上传大小限制
    client_max_body_size 50M;
}
EOF
```

### 7.3 测试并重载配置

```bash
nginx -t              # 测试配置语法
systemctl reload nginx  # 重载配置
```

### 7.4 Nginx 配置详解

| 配置项 | 作用 |
|--------|------|
| `listen 80` | 监听 HTTP 80 端口 |
| `root ...dist` | 指定前端静态文件目录 |
| `try_files` | SPA 路由支持，所有路由都返回 index.html |
| `proxy_pass` | 将 /api 请求转发到后端 Node.js 服务 |
| `proxy_buffering off` | 禁用缓冲，支持 SSE 流式响应 |
| `client_max_body_size` | 限制上传文件大小 |

## 八、配置防火墙

### 8.1 腾讯云控制台配置

1. 进入 **轻量应用服务器** → 选择实例
2. 点击 **防火墙** 标签
3. 添加规则：
   - **TCP 80** (HTTP)
   - **TCP 443** (HTTPS，如需配置)

### 8.2 系统防火墙（可选）

```bash
# 安装 firewalld
dnf install -y firewalld
systemctl start firewalld
systemctl enable firewalld

# 开放端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --reload
```

## 十、部署原理详解

### 10.1 为什么需要 Nginx？

1. **静态文件托管**：Nginx 高效处理静态资源（HTML/CSS/JS/图片）
2. **反向代理**：统一入口，将 API 请求转发到 Node.js
3. **负载均衡**：生产环境可扩展多个后端实例
4. **SSL 终止**：处理 HTTPS 加密/解密

### 10.2 为什么需要 PM2？

1. **进程守护**：崩溃自动重启
2. **日志管理**：统一收集日志
3. **零停机重载**：更新代码无需中断服务
4. **多进程模式**：利用多核 CPU

### 10.3 前后端分离部署流程

```
开发环境:
前端 (localhost:5173) ←→ 后端 (localhost:3001)

生产环境:
                          ┌── 静态文件 (Nginx 直接返回)
用户 → Nginx (:80) ──────┤
                          └── /api/* → Node.js (:3001) → MySQL
```

### 10.4 环境变量的作用

- **开发环境**：`VITE_API_BASE_URL=http://localhost:3001`
- **生产环境**：`VITE_API_BASE_URL=` （空，使用相对路径由 Nginx 代理）

## 十一、问题排查与解决

### 问题 1：NodeSource 安装脚本不兼容

**现象**：
```
Error: This script is intended for RPM-based systems.
```

**原因**：OpenCloudOS 虽基于 CentOS，但脚本识别失败。

**解决**：直接下载 Node.js 二进制包安装。

---

### 问题 2：GitHub 访问超时

**现象**：
```
curl: (28) Connection timed out
```

**原因**：国内网络访问 GitHub 不稳定。

**解决**：
- 使用国内镜像（npmmirror）
- 或使用 SFTP 直接上传代码

---

### 问题 3：API 返回 500 错误

**现象**：
```json
{"code":500,"msg":"获取文章列表失败","data":null}
```

**排查步骤**：
```bash
pm2 logs blog-api --lines 50
```

**常见原因**：
1. 数据库表字段名与代码不匹配
2. 数据库连接配置错误
3. 缺少必要的表

**解决**：检查并修正数据库表结构，确保字段名一致。

---

### 问题 4：登录接口返回 500

**现象**：登录请求返回服务器错误。

**原因**：代码查询 `admin` 表，但只创建了 `users` 表。

**解决**：创建正确的 `admin` 表。

---

### 问题 5：前端请求 localhost 而非服务器地址

**现象**：浏览器网络请求显示 `http://localhost:3001/api/...`

**原因**：前端代码默认值为 `localhost:3001`，构建时未正确设置环境变量。

**解决**：
```bash
# 方法1：修改源码默认值
sed -i "s|'http://localhost:3001'|''|g" src/services/api.js

# 方法2：设置环境变量后重新构建
echo "VITE_API_BASE_URL=http://服务器IP" > .env.production
npm run build
```

---

### 问题 6：Nginx 502 Bad Gateway

**排查**：
```bash
# 检查后端是否运行
pm2 status

# 检查后端端口是否监听
netstat -tlnp | grep 3001
```

**解决**：确保后端服务正常运行。
